# Write Social Media Post

## 語氣與個性

- 像真實的人在說話，不像唸稿或寫公告
- 句子長短有變化，保持節奏感
- 觀點直接說出來，不迂迴鋪墊或過度解釋
- 情緒自然流露，不刻意煽動或誇張
- 不堆砌形容詞，讓事實和例子說話
- **整篇不出現第一人稱**（「我」「我覺得」「我認為」「個人覺得」「筆者」都不寫），判斷直接落下來，觀點從語氣看得出來；動詞用口語的「覺得」而不是「認為本文應當」這種公文體
- **也避免「作者覺得」「文章認為」這種第三人稱代理當主語**，連續用「作者 X」「作者 Y」開段會變成轉述報告。要點名出處用「文章裡」「他寫到」「裡面提到」就好

## 結構與寫法

- 篇幅簡短，重點明確，適合直接貼到群組或社群
- 開頭第一行是用 **粗體** 包裹的標題，點出主題不用套話暖場
- 內文可以用 **粗體** 標出真正重要的詞或句子，保持稀缺感才有效果
- 段落之間保持節奏，不要太密
- **段落開頭句式要多樣化**，不要每段都用「XX 是 YY」「XX 怎麼樣」這種主題宣告開場。可以用具體事實、引述原文、對照差異、動作描述起頭，讓段落不像章節目錄
- **段落開頭忌兩種報告體**：
  - **結構宣告詞**（例如「核心想法是」「重點是」「主要特點包含」「值得關注的是」）：人不會先預告「我要講重點了」
  - **冒號展開句**（例如「判斷很乾脆：……」「驗收分三層：……」「適合的場景：……」「作者的觀點：……」）：「一句話 + 冒號 + 展開」是簡報格式，群組裡沒人這樣講話。例外只有列舉清單前的引導句，或冒號後直接接引述原文
- **用順敘鋪陳，不用冒號切結構**。把「總結句 + 冒號 + 展開」改成順著一句話講完，節奏從報告變成講話
  - 不要寫：「驗收分三層：命令過、眼見為實、需求對清單。」
  - 改寫成：「驗收要分三層走，命令有沒有過、畫面眼見為實、需求逐條對清單。」
- **鋪陳順序：先報導，後觀點**。前面段落只鋪事實、做法、數據，不夾「值得關注」「滿有意思」這類主觀詞；最後一段才收束成判斷
- **結尾觀點要具體**：講「對誰有感」「哪裡微妙」「什麼情況下要保留」這種有判斷力的話，不寫模糊的推薦語
- **不用自我標記觀點**（例如「最後一段的判斷最值得留下來」「這裡的觀點蠻關鍵」「真正的重點在後面」「以下這段最有感」）：主動貼「這是判斷」「這是重點」的標籤反而稀釋判斷力道，觀點要從動詞跟收尾語氣浮出來
- 附上連結（文章、repo 等）時，獨立放在最後一行，不夾在段落裡

## 底線禁令

- 不使用 emoji：讓文字本身承重，不靠符號撐氣氛
- 不使用 em-dash（「——」或「—」）：容易讓句子顯得戲劇化或像翻譯腔
- 不用強推式結尾（例如「這篇很值得讀！」「快去試試看！」「快去看看！」「快去做這件事！」）：讀者自己判斷，不需要被催促
- 不用過於正式或像新聞稿的語氣：這是人寫給人看的，不是公告
